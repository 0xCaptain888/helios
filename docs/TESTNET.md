# Casper Testnet Runbook

Everything needed to take Helios from the local simulation to **live
transactions on Casper Testnet** — the qualification requirement.

## 0. Prerequisites

| Tool | Install |
|---|---|
| Rust + wasm target | `rustup target add wasm32-unknown-unknown` |
| Python 3.10+ | already on your machine |
| `cryptography` pip package | `pip install cryptography` (one package only) |
| 5 funded testnet accounts | auto-generated by the deploy script |

> **No casper-client needed.** All deployment and contract calls are handled
> by `scripts/casper_deploy.py` which uses Python + the `cryptography`
> package for secp256k1/ed25519 signing and speaks directly to the Casper
> JSON-RPC endpoint.

## 1. Build the contracts

```bash
bash scripts/build_contracts.sh
# Expected output: 4 WASM files in contracts/wasm/
```

Contracts are built with:
- **casper-contract v5** + **casper-types v6** (Casper 2.x API)
- `#![no_std]` + `wee_alloc` (no standard library, as required by Casper VM)
- `RUSTFLAGS="-C link-arg=--allow-undefined"` (allows Casper VM host function references)
- Feature flags to select which contract to compile

## 2. Deploy — one command (recommended)

```bash
pip install cryptography          # only dependency
python3 scripts/deploy_helios.py  # generates keys, prompts for faucet, deploys all 4 contracts
```

The script handles everything:
- generates 5 keypairs in `keys/`
- prints faucet URLs and waits for you to fund them
- deploys OracleRegistry → DataMarket → wires them → FundVault → Governance → wires them
- writes `agents/testnet.env` automatically
- produces first on-chain activity (register + attest)

Resume after interruption: `python3 scripts/deploy_helios.py --resume agents/testnet.env`

### Alternative: step-by-step with casper_deploy.py

```bash
# Check key info
python3 scripts/casper_deploy.py pubkey --key "Account 1_secret_key.pem"

# One-click deploy all 4 contracts + wiring
python3 scripts/casper_deploy.py deploy-all --key "Account 1_secret_key.pem"

# Or deploy individually
python3 scripts/casper_deploy.py install --key "Account 1_secret_key.pem" \
  --wasm contracts/wasm/OracleRegistry.wasm --wait

# Call contract entry-points
python3 scripts/casper_deploy.py call --key "Account 1_secret_key.pem" \
  --contract <REGISTRY_HASH> --entry-point register \
  --args "name:string=TBill Oracle" "category:string=rwa" \
         "endpoint:string=http://localhost:8451/quote" "price_motes:u64=2000000000" \
  --wait
```

### Deploy order and init args

1. **OracleRegistry** (no init args)
2. **DataMarket** — init args: `registry_hash` (OracleRegistry contract hash), `fee_bps:u32=250`
3. Wire them: call `set_market(market_hash)` on OracleRegistry
4. **FundVault** — init args: `operator` = deployer's account hash, `governance_hash` = "pending"
5. **Governance** — init args: `proposer` = deployer's account hash, `risk_agent` = risk agent's account hash, `veto_window_ms:u64=90000`
6. Wire: call `set_governance(governance_hash)` on FundVault

## 3. Configure the agents

`deploy_helios.py` writes `agents/testnet.env` automatically. Manual format:

```ini
ORACLE_TBILL_KEY=keys/oracle_tbill/secret_key.pem
ORACLE_GOLD_KEY=keys/oracle_gold/secret_key.pem
ORACLE_REINDEX_KEY=keys/oracle_reindex/secret_key.pem
FUND_AGENT_KEY=keys/fund_agent/secret_key.pem
RISK_AGENT_KEY=keys/risk_agent/secret_key.pem
REGISTRY_HASH=<hex>
MARKET_HASH=<hex>
VAULT_HASH=<hex>
GOV_HASH=<hex>
```

## 4. Produce on-chain activity

```bash
HELIOS_MODE=testnet python3 scripts/testnet_round.py --register --list   # one-time onboarding
HELIOS_MODE=testnet python3 scripts/testnet_round.py --rounds 3          # attest → anchor → propose → finalize → rebalance
```

Each round produces ~10 testnet transactions. Verify on
`https://testnet.cspr.live/account/<account-hash>` and link those URLs in the
submission.

## 5. Reading state for the dashboard

The dashboard's feed is written locally by the agents; for judges who want to
verify directly against the chain:

```bash
# Query contract state via RPC
python3 scripts/casper_deploy.py status
```

Or use CSPR.cloud's REST API with a free API key for indexed queries.

## Troubleshooting

| Symptom | Fix |
|---|---|
| `Wasm preprocessing error: Memory section should exist` | Built with `--import-memory`. Run: `unset RUSTFLAGS && bash scripts/build_contracts.sh` |
| `Out of gas` on deploy | raise payment amount (WASM deploys on testnet often need 200–400 CSPR) |
| `invalid deploy: account has insufficient balance` | hit the faucet again; each account should hold ≥ 1000 test CSPR |
| `undefined symbol: casper_*` | Missing `--allow-undefined` linker flag. Use `RUSTFLAGS="-C link-arg=--allow-undefined"` |
| `can't find crate for 'std'` | Missing `#![no_std]` in lib.rs (already fixed) |
| `version conflict casper-contract` | Stale cache. Run: `rm -rf contracts/target/ && cd contracts && cargo fetch` |
| `Invalid Deploy` | CLType enum values wrong. Fixed in casper_deploy.py (U32=0x04, U64=0x05, U512=0x08) |
| `invalid body hash` | header_hash used account_hash instead of PublicKey serialization. Fixed in casper_deploy.py |

## Supported argument types

| Format | CLType | Example |
|---|---|---|
| `name:string=value` | String | `"name:string=TBill Oracle"` |
| `name:u64=value` | U64 | `"price_motes:u64=2000000000"` |
| `name:u32=value` | U32 | `"fee_bps:u32=250"` |
| `name:u512=value` | U512 | `"amount:u512=400000000000"` |
| `name:bool=value` | Bool | `"active:bool=true"` |
